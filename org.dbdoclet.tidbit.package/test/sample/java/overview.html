Section 1

As implemented, Javadoc requires class {@link org.dbdoclet.test.sample.Beer} and package {@link org.dbdoclet.test.sample} and relies on the java compiler to do its job. Javadoc calls part of javac to compile the declarations, ignoring the member implementation. It builds a rich internal representation of the classes, including the class hierarchy, and "use" relationships, then generates the HTML from that. Javadoc also picks up user-supplied documentation from documentation comments in the source code.
