e-novative DocBook Environment (eDE)

The >e-novative> DocBook Environment (eDE) is a free, ready-to-go integrated DocBook environment for Win32 systems enabling you to create single html pages, multiple html pages and PDF documents from DocBook XML source files.

Enjoy the simplicity of DocBook on Windows

Using DocBook has many advantages over using a regular word processor. A short overview of eDE features is also available. eDE was designed with the average user in mind. No special XML, HTML, CSS, XSL or even Windows knowledge is required to install and use eDE. eDE installs on Win32 systems in a few seconds, does not mess up any registry or put anything into a Windows directory. The document generation is performed by batch files so you only have to enter one command to create output. Since eDE is command-line-oriented, all tasks are particularly easy to automate. eDE currently supports single HTML page, multiple HTML pages and PDF output; HTML Help support will be added soon, other formats like Plain Text, Postscript and Latex might be added in the future.

e-novative DocBook Environment (eDE) News

Oct, 23rd, 2002: eDE forum

David John, working on the eDe manual, and Tab Hockamier, working on the eDE GUI, are on the forum. eDE users, please visit the forum and help us with your comments and suggestions. Chances are good that you'll see your ideas and requests showing up in eDE soon.

Oct, 17th, 2002: eDE GUI on the way

Tab Hockamier, a Phoenix-based Windows and GUI developer with over 15 years of experience has volunteered to create a GUI to eDE. We are looking forward to working together with Tab. Anybody who wants to contribute or comment can contact Tab via the forum at www.e-novative.de/forum. Thanks for joining in, Tab!

Oct 9th, 2002: eDE user forum created

A support and discussion forum for eDE users has been set up to offer the growing number of users a place to ask questions and share information. Have a look at http://www.e-novative.de/forum/.

Oct 8th, 2002: eDE 0.6.0 released

With version 0.6.0, eDE takes a big leap towards where we want to end up: you are in full control of document creation and transformation without requiring any operating system command. All eDE batches were rewritten to be more flexible and have a better error handling. The "official" documentation of all XSL parameters has begun, more parameter documentation is on the way in forthcoming releases. You can easily create document-specific stylesheets and using the new automated backup and restore mechanism you don´t have to worry about individual files, but can think of an eDE DocBook document as a whole.

We strongly recommend all users to upgrade to eDE 0.6.0. since working with eDE 0.6.0 has become much easier, and some 0.5.x bugs are fixed.

View the full changelog here.

Oct 7th, 2002: eDE User Manual on the way

David John, a London-based technical writer, has volunteered to create a comprehensive user documentation for eDE. The document will focus on customizing eDE to user needs. e-novative will help David out with as much support and information as possible, and we are already eager to see the first versions of David's documentation. Thanks for joining in, David!

Oct 3rd, 2002: eDE 0.5.3 released

eDE 0.5.3 features fully automated deployment of all output formats as zip files along with some minor changes and improvements. You can now define and use XML entities, either local to one document or global to all documents. This lets you include repeating text in one or multiple documents in a snap. A section on upgrading eDE has been included in the documentation (full changelog).
